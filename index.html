CNN-Based Detection and Classification of Lung and Colon Cancer (2025)

Developed an automated deep learning-based diagnostic system for detecting and classifying lung and colon cancer from histopathological images. The system utilized Convolutional Neural Networks (CNNs), specifically DenseNet-121 and ResNet-50 architectures, to accurately differentiate between benign and malignant tissue samples across five classes. The workflow included image preprocessing (resizing, Gaussian filtering, contrast enhancement), Modified Fuzzy C-Means (MFCM) segmentation for isolating regions of interest, and texture-based feature extraction using Gray Level Co-occurrence Matrix (GLCM). Particle Swarm Optimization (PSO) was applied for optimal feature selection, improving classification efficiency. A hybrid classification approach combining Multi-Class SVM and deep learning models was implemented and evaluated using metrics such as accuracy, sensitivity, specificity, F-score, Dice coefficient, and Jaccard index. The system was integrated with a user-friendly MATLAB GUI to enable real-time image upload and automated cancer prediction, supporting medical professionals in faster and more reliable diagnosis.

Fire Detection using Haar Cascade Classifier

The Fire Detection Using Haar Cascade Classifier project is a real-time computer visionâ€“based safety system developed to detect fire using the built-in camera of PC. The system is implemented in Python using the OpenCV and is based on the Haar Cascade algorithm proposed by Paul Viola and Michael Jones. The live video feed from the inbuilt webcam is processed frame by frame to identify fire regions, and when fire is detected, the system highlights the area and displays a notification on the screen. This project demonstrates the integration of computer vision with embedded systems to create a practical, real-time fire safety monitoring solution.

Buffer Overflow Demonstration in C (2026)

Developed a C-based application intentionally designed with a stack buffer overflow vulnerability to demonstrate insecure memory handling practices and the risks of improper bounds checking. The program utilizes unsafe input functions such as gets() and unchecked scanf to illustrate how unvalidated user input can overwrite adjacent memory regions, corrupt stack frames, and potentially alter program execution flow. Through this project, I analyzed stack memory layout, function call behavior, and the impact of overwriting local variables and return addresses. The implementation was tested in a controlled environment to observe segmentation faults and abnormal execution patterns, providing practical insight into how low-level vulnerabilities are exploited and reinforcing the importance of secure coding practices, input validation, and modern mitigation techniques in C programming.
